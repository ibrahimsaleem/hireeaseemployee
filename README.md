# Aplyease Employee Portal

A **completely independent** full-stack app for Aplyease employees.
One single service — Express serves both the React frontend and the API from the same URL.

```
employee-portal/
  server/          ← Express API (Node.js + Drizzle ORM)
  src/             ← React frontend (Vite + Tailwind)
  dist/            ← built frontend (auto-generated by npm run build)
  package.json     ← single package.json for both
```

---

## Architecture

```
Employees → https://your-app.onrender.com
               │
               ▼ (single Render service)
         Express server
           ├─ /api/*   → REST API (JWT auth)
           └─ /*       → serves React SPA from dist/
               │
               ▼ SQL
         Supabase PostgreSQL (shared with main app)
```

- No separate Vercel/Netlify needed — the Express server hosts everything
- No dependency on the main Aplyease backend
- Employees log in with their existing credentials (EMPLOYEE or ADMIN role)

## Local Development

### 1. Install dependencies
```bash
cd employee-portal
npm install
```

### 2. Configure environment
Copy `.env.example` to `.env` and fill in your values:
```bash
cp .env.example .env
```

Required variables in `.env`:
```env
DATABASE_URL=postgresql://...supabase...
NODE_TLS_REJECT_UNAUTHORIZED=0
SESSION_SECRET=any-long-random-string
GEMINI_API_KEY=your-key
GEMINI_PUBLIC_MODEL=gemini-2.5-flash
PORT=3001
```

`.env.local` (already present) points the Vite dev server at the local API:
```env
VITE_API_URL=http://localhost:3001
```

### 3. Start everything (one command)
```bash
npm run dev
```
This starts **both** the backend (port 3001) and the frontend dev server (port 5173) concurrently.

### 4. Open the portal
Go to `http://localhost:5173` and log in with any EMPLOYEE or ADMIN account.

## Deploy to Production (Render — single service)

1. Push to GitHub

2. Create a new **Web Service** on [render.com](https://render.com):
   - **Root Directory**: `employee-portal`
   - **Build Command**: `npm install && npm run build`
   - **Start Command**: `npm start`
   - **Runtime**: Node

3. Set environment variables in the Render dashboard:
   ```
   DATABASE_URL                   = your Supabase connection string
   SESSION_SECRET                 = any long random string
   GEMINI_API_KEY                 = your Gemini key
   GEMINI_API_KEY_2               = fallback Gemini key (optional)
   GEMINI_PUBLIC_MODEL            = gemini-2.5-flash
   NODE_TLS_REJECT_UNAUTHORIZED   = 0
   NODE_ENV                       = production
   ```
   > Do NOT set `PORT` — Render injects it automatically.
   > Do NOT set `VITE_API_URL` — in production the frontend calls relative `/api/*` paths on the same origin.

4. Your portal will be live at `https://aplyease-employee-portal.onrender.com`
   (or whatever name you choose in Render)

## How it works in production

`npm run build` builds the React app to `dist/`.
`npm start` runs Express which:
- Serves `dist/` as static files
- Handles `/api/*` routes with the REST API
- Returns `dist/index.html` for any other path (SPA fallback)

No CORS configuration needed in production since the frontend and API share the same domain.

## API Routes

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/auth/login` | Login (EMPLOYEE/ADMIN only) |
| GET | `/api/auth/user` | Get current user |
| POST | `/api/auth/logout` | Logout |
| GET | `/api/clients` | List assigned clients |
| GET | `/api/client-profiles/:userId` | Get client profile |
| GET | `/api/resume-profiles/:clientId` | List resume profiles |
| GET | `/api/applications` | List applications (paginated) |
| POST | `/api/applications` | Submit new application |
| PATCH | `/api/applications/:id` | Update application status |
| GET | `/api/stats/employee/:id` | Employee stats |
| GET | `/api/payment-transactions/:clientId` | Payment history |
| POST | `/api/generate-resume/:clientId` | AI tailor resume |
| POST | `/api/generate-pdf` | Compile LaTeX → PDF |
| GET | `/api/health` | Health check |
